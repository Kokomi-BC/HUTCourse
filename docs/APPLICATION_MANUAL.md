## 2. 产品简介

HUTCourse 是面向高校校园场景的智能助手应用，聚焦以下高频需求：

1. 课表查看与同步。
2. 日程新增、编辑、重复提醒管理。
3. AI 对话式查询（课表、日程、导航、空教室、联网检索）。
4. 校内位置识别与导航。
5. 天气、记忆、数据导入导出等辅助能力。

适用人群：在校学生、辅导员、校园活动组织者。

## 3. 安装与启动

## 3.1 安装要求

1. Android 8.0 及以上（minSdk 26）。
2. 建议开启网络连接。
3. 建议允许定位权限（用于距离估算与导航）。

## 3.2 首次启动

首次进入应用后建议按以下顺序配置：

1. 进入“设置”。
2. 在“账号与教务同步”中登录教务并同步课表。
3. 在“AI 接入与技能”中配置模型 API 信息。
4. 根据喜好在“显示设置”中调整主题色、字体大小。

## 4. 页面与功能说明

## 4.1 主页面导航

底部导航默认包含 4 个主入口：

1. 今日：展示今日课程、日程概览、天气。
2. AI：进入智能对话页。
3. 课表：按周查看课程网格。
4. 我的：展示个人信息与设置入口。

## 4.2 课表页

### 可执行操作

1. 查看当前周课表，支持切换周次。
2. 查看课程详情（教师、地点、周次、节次）。
3. 在课程详情中编辑教师、地点、周次、颜色等信息。
4. 根据当前位置显示课程地点距离提示（需定位权限）。

### 使用建议

1. 若周次不准确，请在设置中配置“开学日期”。
2. 若课程缺失，返回“账号与教务同步”执行刷新。

## 4.3 今日页

### 内容组成

1. 今日课程时间线。
2. 今日/近期日程摘要。
3. 天元区天气（含刷新与折叠）。
4. 下一节课提醒卡片。

### 操作说明

1. 点击天气刷新图标可重新拉取天气。
2. 点击日程入口可跳转日程总览页。
3. 点击任意日程可直接编辑。

## 4.4 日程总览页

### 功能

1. 按状态分组：进行中 / 将要开始 / 已结束。
2. 支持手动新增与 AI 生成两种新增方式。
3. 支持编辑标题、描述、地点、日期、时段、优先级、重复规则。
4. 支持删除日程。

### 重复规则说明

1. 不重复：仅在基准日期生效。
2. 每天：每日生效。
3. 每周：按基准日期的星期循环。
4. 每月固定日：按日期日号循环。

短月策略：

1. 短月跳过：当月无该日号则不触发。
2. 短月改月底：当月无该日号则移动到月末。

## 4.5 AI 对话页

### 主要能力

1. 支持文字对话。
2. 支持图文输入（最多 4 张图片）。
3. 支持多模型切换。
4. 支持会话历史查看、切换与新建。
5. 支持工具调用回执与系统卡片交互。

### 系统卡片

AI 在以下场景会显示卡片：

1. 教务登录失效：引导跳转登录并自动续跑。
2. AI 配置缺失：引导跳转 AI 设置页面。
3. 导航结果：点击后拉起高德地图导航。

### 内置快捷命令（高级）

1. `/memories`：读取记忆。
2. `/memory 内容`：新增记忆。
3. `/memory-edit 序号 内容`：修改记忆。
4. `/memory-del 序号或关键词`：删除记忆。
5. `/memory-clear`：清空记忆。
6. `/cmd 命令`：手动执行技能命令。

## 4.6 设置页

设置主页包含 4 个子页：

1. 账号与教务同步
2. 显示设置
3. 数据管理
4. AI 接入与技能

### 账号与教务同步

1. 设置开学日期。
2. 打开教务登录页。
3. 从教务刷新课表。
4. 清除本地课表（保留登录）。
5. 退出教务登录。

### 显示设置

1. 开关课表网格线。
2. 设置课表主题色（含十六进制自定义）。
3. 调整课表字体比例（85%-130%）。
4. 编辑课程颜色与课程基础信息。

### 数据管理

1. 课表导出到剪贴板。
2. 从剪贴板导入课表。
3. 导出 Cookie 到剪贴板。
4. 从剪贴板导入 Cookie。

### AI 接入与技能

1. 新增、编辑、删除模型。
2. 拖拽调整模型优先级。
3. 设置模型 provider、baseUrl、apiKey、modelName。
4. 开关技能：memory/course/navigation/classroom/agenda/web_search。
5. 配置 Tavily（联网检索）API。

## 5. 典型使用场景

## 5.1 场景一：第一次使用同步课表

1. 打开“设置 -> 账号与教务同步”。
2. 点击“打开教务系统”完成登录。
3. 点击“从教务同步最新课表”。
4. 返回课表页检查周次与课程数量。

## 5.2 场景二：一句话查空教室

示例提问：

- “明天上午 10 点到 11 点有空教室吗？”

系统行为：

1. AI 解析日期和时段。
2. 调用空教室技能。
3. 返回可用公共教室列表。
4. 若登录失效则弹出登录卡片，登录后可继续查询。

## 5.3 场景三：自然语言创建日程

示例提问：

- “帮我创建一个每周三下午四点在图书馆自习的日程。”

系统行为：

1. 解析标题、时间、地点、重复规则。
2. 调用 `agenda.create` 写入本地数据库。
3. 在今日页/日程页可见新建结果。

## 5.4 场景四：校内导航

示例提问：

- “我现在在哪？”
- “怎么去崇慧楼？”

系统行为：

1. 定位并语义化输出当前位置。
2. 估算路程或生成导航卡片。
3. 用户点击卡片后拉起高德地图。

## 6. 常见问题（FAQ）

## 6.1 课表刷新失败怎么办？

1. 检查是否已登录教务系统。
2. 在“账号与教务同步”重新登录并刷新。
3. 若仍失败，稍后重试（可能是教务系统网络波动）。

## 6.2 AI 无法回答或提示模型未配置？

1. 进入“AI 接入与技能”。
2. 检查是否已配置模型名与 API Key。
3. 确认至少存在一个可用模型。

## 6.3 空教室查询要求登录吗？

需要。空教室查询依赖教务系统会话，登录失效后请按卡片提示重新登录。

## 6.4 定位不准确怎么办？

1. 确保已授予定位权限。
2. 打开系统定位服务（GPS/Wi-Fi）。
3. 在课程详情中点击刷新位置按钮。

## 6.5 联网搜索无结果怎么办？

1. 确认已开启 `web_search` 技能。
2. 确认 Tavily API Key 已配置。
3. 尝试缩短检索关键词并重试。

## 7. 权限与隐私说明

## 7.1 权限用途

1. 网络权限：用于教务同步、AI 请求、天气与联网检索。
2. 定位权限：用于校内位置识别、距离估算、导航辅助。

## 7.2 数据说明

1. 课表、日程、模型配置等数据存储于本机。
2. AI 对话内容会发送至配置的模型服务。
3. Cookie 仅用于教务会话，不用于其他业务。

---

## 8. 异常处理与排障建议

1. 网络异常：检查网络后重试。
2. 页面卡顿：关闭并重启应用，必要时清理缓存。
3. 同步异常：尝试“退出登录 -> 重新登录 -> 重新同步”。
4. 模型异常：切换备用模型或检查接口 Base URL。
